# Pencil vs RadFlow Architecture Comparison

## Problem

RadFlow is a **Design System Manager for LLM CLI tools** with a clipboard-first philosophy (context engineering, not direct writes). Pencil.dev emerged as a mature design tool with overlapping goals but different philosophy (design becomes code via translation).

Before completing RadFlow's Component Canvas and Comment Mode features, we need a comprehensive comparison to:

1. Understand what Pencil does well (component display, streaming, MCP integration)
2. Evaluate if RadFlow's architecture decisions (CSS scale, two-tier tokens, context-design) are still sound
3. Identify patterns worth adopting vs. maintaining differentiation
4. Create a roadmap that incorporates the best of both approaches

**Key philosophical difference:**
- Pencil: "Design becomes code" (translation layer, direct writes)
- RadFlow: "Design IS code" (no abstraction, clipboard-first context bundling)
