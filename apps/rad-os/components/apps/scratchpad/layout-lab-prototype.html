Design is not just what it looks like and feels like. Design is how it works. Every pixel, every token, every spacing decision carries intention. The pretext engine gives us direct control over where each glyph lands on the canvas, bypassing the browser's layout engine entirely.

When text flows around obstacles, wraps across columns, and responds to container width changes, it does so through explicit measurement and placement. No CSS reflow. No layout thrashing. Just math and a canvas context.

The fluid type scale maps seven tiers from sm through 4xl. Each tier defines a clamp curve: a minimum floor, a base offset, a scaling coefficient, and a maximum ceiling. As the container grows, type sizes interpolate smoothly between these bounds.

Columns are not CSS multi-column. They are computed regions. The engine measures each word, finds optimal break points using Knuth-Plass dynamic programming, and distributes lines across column slots. Gutter rules are drawn at exact pixel positions between columns.

Hyphenation uses a syllable dictionary with fallback heuristics. Soft hyphens are injected at valid break points, and the layout algorithm considers hyphen penalties when choosing line breaks. The result is tighter, more even text color than browser hyphens can achieve.

Obstacles are geometric primitives registered on the layout surface. The engine scans each line band against the obstacle hull, carves available text slots, and flows words into the remaining space. Circle obstacles use polygon approximation for smooth wrap contours.
