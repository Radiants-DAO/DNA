01 / 14

The Old Pipeline

Designer → Figma → Spec → Developer → "That's not what I meant" → Repeat

The paradigm of direct-write is over. Designers were tweaking values in a database with sliders for you to then abstract into actual code.

Now designers use LLMs to code. And you're using LLMs to code design.

02 / 14

The Collapse

The gap between "I know what I want" and "I have it" went from weeks to minutes.

  • 60 sec Prompt → working UI
  • 30 sec Screenshot → revised UI
  • ∞ HTML prototype → rich CSS to inform Expo / React / any app rebuilds
03 / 14

Thesis

Design is no longer about making choices. It's about generating them.

The best design you'll ship this weekend isn't the one you planned. It's the one you discovered. Create variants, try weird shit on a new branch. "Whoa, that's cool."

04 / 14

Design Is Jazz Now

Coltrane didn't compose note by note. He improvised within constraints.

Your constraints are the lego sets you have in your lego box.

Have a React web component library already built? That can inform the styling of your Expo library. Everything is context now — design systems are file systems, not brand manuals.

05 / 14

The Feedback Loop

The only workflow you need

  1. Generate — describe it, get a prototype
  2. Experience — click through it, feel it
  3. Inspect — screenshot + rich LLM inspection
  4. Critique — "this isn't right, here's the component, edit, use X skill"
  5. Regenerate

↻ Repeat until it feels right.

06 / 14

Prototypes Can Be Anything Now

The format doesn't matter

HTML page with mock data
Somebody else's website or app
React artifact
Figma prototype
Variant screens
An annotated screenshot

Can you experience the app? If yes, you have enough to iterate.

07 / 14

The Critique Prompt

The most powerful sentence in your toolkit

"This is not quite right. Review this HTML component and this screenshot, and edit it to match more closely."
Screenshot + Component ref + Natural language = Precise iteration

That's the whole loop.

08 / 14

Branching, Not Perfecting

Don't iterate on one thing. Branch into many.

Same app, different UI libraries → different results
Same screen, different layouts → different results
Take the nav from one
The cards from another
The palette from a third

Design as remix.
