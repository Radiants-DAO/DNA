# fn-13: Component Browser Enhancement

## Overview

Transform the Component Browser from a static list into a functional preview and editing environment. Currently components show in a list but cannot be previewed or edited.

**Goal:** Enable live component preview, props exploration, and visual style editing.

**Priority:** P0/P1 - Core editing feature

**Estimated Hours:** 35-50h

**Dependencies:** fn-9 (write commands for style editing)

## Task Breakdown

### Phase 1: Live Preview (P0)

**fn-13.1: Create Preview Infrastructure** (4-5h)
- Create preview route in target project template
- Handle component isolation
- Configure iframe communication
- Setup hot reload support

**fn-13.2: Build Preview Panel** (4-5h)
- Create `ComponentPreview.tsx`
- Embed iframe for live render
- Handle loading/error states
- Responsive preview sizing

**fn-13.3: Implement Props Passing** (3-4h)
- Send props via postMessage
- Handle prop type serialization
- Support callback props (mock)
- Real-time prop updates

### Phase 2: Props System (P1)

**fn-13.4: Implement Prop Detection** (4-6h)
- Parse TypeScript interfaces
- Extract prop names and types
- Detect required vs optional
- Handle union types

**fn-13.5: Build Props Playground** (6-8h)
- Create `PropsPlayground.tsx`
- Generate inputs per prop type
- Live update preview
- Save prop configurations

**fn-13.6: Create Props Reference Panel** (3-4h)
- Show prop documentation
- Display default values
- Link to source code
- Type information

### Phase 3: Variants & Sizes (P1)

**fn-13.7: Implement Variant Detection** (3-4h)
- Detect variant props
- Parse variant values
- Group by variant type
- Handle compound variants

**fn-13.8: Build Variant Display** (2-3h)
- Show variant matrix
- Toggle between variants
- Preview all variants
- Variant comparison view

**fn-13.9: Implement Size Detection** (2-3h)
- Detect size props
- Common size patterns (sm, md, lg)
- Show size options
- Size comparison view

### Phase 4: Component Manifest (P1)

**fn-13.10: Support Component Manifest** (4-6h)
- Define manifest schema
- Parse `component.json` files
- Override detected metadata
- Enable custom categorization

## Technical Notes

### Preview Architecture
```
┌─────────────────────┐     ┌──────────────────────┐
│   RadFlow Tauri     │     │   Target Project     │
│                     │     │                      │
│  ┌───────────────┐  │     │  ┌────────────────┐  │
│  │ ComponentPrev │──┼─────┼──│ /preview/:id   │  │
│  │ (iframe)      │  │     │  │ Isolated render│  │
│  └───────────────┘  │     │  └────────────────┘  │
│                     │     │                      │
│  postMessage ←────────────────→ postMessage     │
└─────────────────────┘     └──────────────────────┘
```

### Prop Type to Input Mapping
```typescript
const propInputMap = {
  string: 'TextInput',
  number: 'NumberInput',
  boolean: 'Checkbox',
  enum: 'Select',
  union: 'RadioGroup',
  function: 'MockButton',
};
```

### Component Manifest Schema
```json
{
  "name": "Button",
  "category": "Actions",
  "description": "Primary action button",
  "variants": ["primary", "secondary", "ghost"],
  "sizes": ["sm", "md", "lg"],
  "defaultProps": {
    "variant": "primary",
    "size": "md"
  }
}
```
