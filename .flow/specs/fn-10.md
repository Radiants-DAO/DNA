# fn-10: Theme System

## Overview

Implement the complete theme system that enables multi-brand workflows. Currently the editor is hardcoded to a single theme; this epic adds theme discovery, switching, and management.

**Goal:** Enable "editor is console, theme is game" architecture per spec.

**Priority:** P0 - Critical (architectural foundation)

**Estimated Hours:** 25-35h

**Dependencies:** None (can run parallel to fn-9)

## Task Breakdown

### Phase 1: Theme Foundation (P0)

**fn-10.1: Add Theme Type Definitions** (1-2h)
- Add `ThemeInfo`, `ThemeConfig`, `ColorMode` types to `types/mod.rs`
- Add specta derive macros
- Generate TypeScript bindings

**fn-10.2: Create Theme Module** (2-3h)
- Create `src-tauri/src/commands/theme.rs`
- Add to `mod.rs`
- Register commands with Tauri

**fn-10.3: Implement Theme Discovery** (3-4h)
- Add `list_themes()` command
- Scan workspace for radflow-theme packages
- Parse package.json for theme metadata
- Return sorted list

**fn-10.4: Implement Theme Switching** (4-6h)
- Add `switch_theme(theme_id)` command
- Update app state
- Load theme's CSS variables
- Notify frontend of change

**fn-10.5: Decouple Editor from Single Theme** (2-3h)
- Remove hardcoded theme references
- Use theme context throughout UI
- Ensure all components respect active theme

### Phase 2: Theme Management (P1)

**fn-10.6: Build Theme Management UI** (4-6h)
- Create `ThemePanel.tsx`
- Show theme list with previews
- Enable theme switching
- Show active theme indicator

**fn-10.7: Implement Color Mode Toggle** (2-3h)
- Add light/dark toggle to UI
- Wire to theme's color modes
- Persist preference
- Respect system preference

**fn-10.8: Add Theme Persistence** (2-3h)
- Save selected theme to app config
- Restore on launch
- Handle missing theme gracefully

**fn-10.9: Implement Theme Preview Mode** (3-4h)
- Preview theme without switching
- Show side-by-side comparison
- Revert on cancel

**fn-10.10: Add Theme Validation** (3-4h)
- Validate theme structure on discovery
- Check required tokens exist
- Warn about missing/invalid values

## Technical Notes

### Theme Types
```rust
// src-tauri/src/types/mod.rs
#[derive(Serialize, Deserialize, specta::Type)]
pub struct ThemeInfo {
    pub id: String,
    pub name: String,
    pub description: Option<String>,
    pub path: PathBuf,
    pub color_modes: Vec<String>,
    pub token_count: usize,
}

#[derive(Serialize, Deserialize, specta::Type)]
pub enum ColorMode {
    Light,
    Dark,
    System,
}
```

### Theme Discovery
- Look for packages with `"radflow-theme": true` in package.json
- Parse `tokens.css` or `variables.css` for token count
- Check for `light.css` and `dark.css` for color modes
