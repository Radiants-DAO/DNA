# fn-16.5 Compare RadTools Architecture vs codebase

## Description
Compare the RadTools Architecture document (`~/Downloads/radtools-architecture.md`) against the current codebase and document discrepancies.

## Architecture Doc Claims

| Feature | Spec Says | Status |
|---------|-----------|--------|
| Component Canvas | Shadow DOM isolation | NOT BUILT - future scope |
| Page Builder | Lightweight server or static | BUILT - uses iframe + localhost |
| Live Preview | Optional Vite sidecar | NOT BUILT |
| SWC/OXC parsing | Rust-native JSX transform | BUILT - components.rs uses SWC |
| Source mapping | data-rad-source attributes | BUILT - component scanning |
| Copy-on-import | shadcn-style components | BUILT - theme-rad-os has this pattern |

## Key Discrepancies to Document

1. **Shadow DOM vs Iframe**: Spec describes Shadow DOM for Component Canvas, but current implementation uses iframe (Page Builder pattern). Both are valid for different modes.

2. **Three modes vs two**: Spec describes Component Canvas, Page Builder, Live Preview. Current implementation has Page Builder working, Component Canvas is future.

3. **Source mapping**: Spec says `data-rad-source`, implementation may use `data-radflow-id` - verify attribute names.
