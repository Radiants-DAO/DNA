# fn-3-46x.5 Compare canvas rendering approaches with recommendation

## Description
Deep comparison of canvas rendering approaches with clear **CSS vs WASM** focus.

### Approaches to Compare

**1. CSS transform scale()**
- Research article's recommendation
- GPU-composited, no main thread blocking
- ~1.9 MB GPU memory per 800×600 preview
- Automatic updates, no capture step
- RadFlow's current direction

**2. WASM Canvas**
- Pencil's approach (pencil.wasm - 7.5MB)
- More control over rendering
- WebGL/WebGPU considerations
- Implementation complexity

**3. Shadow DOM Isolation**
- RadFlow vault spec for Component Canvas
- Style encapsulation benefits
- Browser support and limitations

**4. Iframe Isolation**
- Storybook/Sandpack pattern
- Security model
- Communication overhead

### Analysis Dimensions

For each approach:
- Performance characteristics
- Implementation complexity
- Memory footprint
- Scalability (100+ components)
- Hot reload integration
- Developer experience

### Output

Clear recommendation for RadFlow with rationale based on:
- RadFlow's clipboard-first philosophy
- Existing Page Builder iframe architecture
- Component Canvas planned Shadow DOM approach
- Research article findings
